# AGENTS.md — правила для ИИ-агентов на PromptBeats

Этот проект собирают три человека, каждый со своим ИИ-агентом. Чтобы куски состыковались и никто не переделывал
чужое, любой агент, работающий в репозитории, **обязан** соблюдать правила ниже.

## 0. Контекст за 30 секунд
PromptBeats: веб-морда с чатом → пользователь пишет пожелание → LLM превращает его в **Song JSON** →
движок **Tone.js** проигрывает JSON в браузере. Правки трека — тем же путём (промт + текущий JSON → новый JSON).
Полный замысел — в `PLAN.md`.

## 1. Контракты — это закон
- Единственный источник правды по интерфейсам — **`CONTRACTS.md`** и **`song.schema.json`**.
- Кодь строго против них. Не выдумывай поля, эндпоинты и сигнатуры, которых там нет.
- **Нельзя** менять форму Song JSON, HTTP API или интерфейс Player, не обновив `CONTRACTS.md`
  (+ `song.schema.json` при изменении данных) и не записав это в `IMPLEMENTATION_LOG.md` как изменение контракта.
- Не уверен в контракте — спрашивай владельца компонента, а не додумывай.

## 2. Границы компонентов (не залезай в чужое)
| Компонент | Владелец | Папка | Наружу отдаёт |
|-----------|----------|-------|---------------|
| Фронт + чат-UX | A | `frontend/` | UI, состояние `currentSong` |
| Бэк + LLM-шлюз | B | `backend/` | `POST /api/compose`, `GET /api/catalog` |
| Аудио-движок | C | `frontend/src/player/` | модуль `createPlayer()` |

Работай в своей папке. Нужен результат чужого компонента, которого ещё нет — **мокай через `sample-song.json`**,
не лезь его писать за коллегу.

## 3. Дисциплина логирования (главное!)
После каждой законченной единицы работы агент **дописывает запись в `IMPLEMENTATION_LOG.md`** по шаблону оттуда.
Запись должна отвечать на три вопроса: **что сделано, где лежит, как этим пользоваться** (импорт/вызов/эндпоинт + пример).
Это позволяет другим агентам подхватывать готовое, а не переоткрывать.

Записывай также: отклонения от контракта, известные баги, что осталось (TODO). Не переписывай старые записи —
добавляй новые сверху (журнал, а не документ).

## 4. Определение «готово» (Definition of Done) по компонентам
- **B (бэк):** `POST /api/compose` принимает `{prompt, song?}`, возвращает валидный по схеме Song + `message`;
  валидация ответа LLM + ретрай реализованы; `GET /api/catalog` отдаёт каталог. Запись в логе есть.
- **C (движок):** `createPlayer().load(sample-song.json)` играет звук; `load` идемпотентен; неизвестный `sound`
  не роняет; события `step/ready/error` эмитятся. Запись в логе есть.
- **A (фронт):** чат → вызов `/api/compose` → `player.load(res.song)` → play; история правок видна; грид дорожек
  рисуется из `song.tracks[].events`. Запись в логе есть.

## 5. Практики
- Маленькие коммиты с внятным сообщением. Одна фича — один PR/ветка, если работаете через ветки.
- Перед пушем прогони то, что можно прогнать (валидация JSON по схеме, сборка, быстрый smoke-тест).
- Секреты (LLM API-ключи) — только в `.env`, никогда в код и не в коммит. Добавь `.env` в `.gitignore`.
- Общие фикстуры лежат в корне (`sample-song.json`). Не плоди дубликаты.

## 6. Быстрая валидация Song JSON
```bash
python3 -c "import json,jsonschema;jsonschema.validate(json.load(open('YOUR.json')),json.load(open('song.schema.json')));print('OK')"
```
