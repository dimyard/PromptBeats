# PromptBeats — план хакатона (8 часов, 3 человека)

## Идея одним абзацем
Веб-морда с чатом: пользователь пишет пожелание ("сделай медленный меланхоличный лоу-фай бит, 75 BPM"),
LLM превращает это в структурированный **Song JSON**, движок **Tone.js** проигрывает JSON в браузере семплами
и синтами. Пользователь слушает и правит текстом ("добавь хэты", "сделай бас глубже", "убери второй куплет") —
LLM патчит тот же JSON, плеер перезагружается. Экспорт в WAV — по возможности.

## Ключевые решения (не пересматривать во время хакатона)
- **Движок: Tone.js** (не openDAW). Причина: 8 часов, нет аудио-спеца. Tone.js = чистый браузер, MIT,
  ноты/паттерны/семплы/синты из коробки, мгновенная перестройка при правках. openDAW слишком тяжёл для срока.
- **Единый контракт — Song JSON.** Всё общение фронт↔бэк↔LLM↔плеер идёт через одну схему. Кто-то трогает схему —
  говорит вслух всей команде.
- **Редактирование = тот же путь, что генерация.** LLM получает текущий JSON + текст правки → возвращает новый JSON.
  Никакого отдельного механизма правок не пишем.
- **Стек:** фронт React + Tone.js; бэк Node/Express (или FastAPI) как тонкий шлюз к LLM. MCP — опционально
  (см. "Растяжка"), на MVP не критично.

## Song JSON — контракт (v1, минимальный)
```json
{
  "version": 1,
  "title": "lofi sketch",
  "bpm": 75,
  "key": "A minor",
  "bars": 8,
  "tracks": [
    {
      "id": "drums",
      "instrument": "sampler",
      "sound": "lofi_kit",
      "pattern": [
        { "step": 0,  "note": "C2", "vel": 0.9 },
        { "step": 4,  "note": "D2", "vel": 0.6 },
        { "step": 8,  "note": "C2", "vel": 0.9 }
      ]
    },
    {
      "id": "bass",
      "instrument": "synth",
      "sound": "sine_bass",
      "notes": [
        { "start": 0, "dur": 2, "note": "A1", "vel": 0.8 },
        { "start": 2, "dur": 2, "note": "F1", "vel": 0.8 }
      ]
    }
  ]
}
```
Правила: время в шагах (16 шагов = такт) для ударных и в долях для мелодических дорожек; `sound` — имя из
фиксированного каталога пресетов (см. ниже). LLM обязан выбирать `sound` только из каталога — это ловим валидатором.

## Каталог звуков (фиксируем в начале, ~10 штук)
Синты Tone.js (без ассетов): `sine_bass`, `saw_lead`, `soft_pad`, `pluck`, `fm_bell`.
Семпл-киты (пара бесплатных, лицензионно чистых наборов): `lofi_kit`, `house_kit`, `trap_kit`.
Меньше — надёжнее. Каталог = enum в JSON-схеме, по нему валидируем ответ LLM.

## Распределение на троих
Каждый владеет своим слоем, но контракт (Song JSON) общий. Договоритесь о схеме в первые 30 минут,
дальше работаете параллельно на моках.

### Человек A — Фронт + чат-UX (владелец веб-морды)
- Экран: поле чата слева, "плеер + список дорожек" справа.
- Кнопки: Play / Stop, BPM-индикатор, история сообщений (чтобы видеть итерации правок).
- Отправляет на бэк `{ prompt, currentSong }`, получает новый `Song JSON`, кладёт в состояние.
- Показывает дорожки/паттерн визуально (простой step-grid — достаточно кружочков).
- Пока бэка нет — работает на замоканном JSON из файла.

### Человек B — Бэкенд-шлюз + LLM (владелец промт→JSON)
- Эндпоинты: `POST /generate` ({prompt}) и `POST /edit` ({prompt, currentSong}). Можно объединить в один.
- Системный промт для LLM: описание схемы + каталог звуков + требование "верни ТОЛЬКО валидный JSON".
- **Валидатор** ответа (JSON Schema / zod): проверяет структуру, чинит/реджектит мусор, ретрай при ошибке.
- Few-shot примеры в промте: 2-3 пары "пожелание → JSON" и одна пара "правка → JSON".
- Отдаёт фронту чистый Song JSON.

### Человек C — Аудио-движок на Tone.js (владелец плеера)
- Модуль `player(song)`: строит Tone.Transport, инструменты по `sound`, расставляет ноты/паттерн, играет луп.
- Реестр звуков: маппинг имя→Tone-инструмент (синты) и загрузка семпл-китов.
- `updateSong(newSong)` — корректно останавливает и пересобирает граф без утечек/дребезга.
- Растяжка: рендер в WAV (Tone.Offline) для кнопки "Скачать".
- Пока бэка/фронта нет — гоняет на том же замоканном JSON.

Общий артефакт дня-ноль: **`sample-song.json`** в репозитории. A и C кодят против него, пока B поднимает LLM.

## Почасовые вехи (~8 часов)
- **0:00–0:30** Все вместе: финализируем Song JSON и каталог звуков. Создаём `sample-song.json`. Заводим репо/скелет.
- **0:30–2:30** Параллельно на моках:
  - A: чат-UI + состояние + отрисовка дорожек из моканого JSON.
  - B: `/generate` возвращает захардкоженный JSON (заглушка), затем подключает LLM.
  - C: `player(sample-song.json)` играет звук в браузере.
- **2:30–3:00** Первая стыковка: фронт зовёт бэк, получает JSON, передаёт в плеер. "Промт → звук" вживую.
- **3:00–5:00**:
  - B: доводит промт+валидатор, добавляет `/edit` (правки), few-shot.
  - C: реестр из ~10 звуков, чистая пересборка при правке.
  - A: история правок в чате, состояние "текущий трек".
- **5:00–6:30** Интеграция цикла редактирования: правка текстом → патч JSON → перезагрузка плеера.
- **6:30–7:30** Полировка демо: 2-3 заготовленных сценария, стилизация UI, обработка ошибок LLM (ретрай/тост).
- **7:30–8:00** Прогон демо-сценария 2 раза. Заморозка. Бэкап рабочей ветки.

## Демо-сценарий (репетировать заранее)
1. "Сделай спокойный лоу-фай бит, 75 BPM, минор" → играет.
2. "Добавь мягкий пэд на фоне" → перестроилось.
3. "Ускорь до 90 и сделай бас громче" → слышно разницу.
4. (если успели) "Скачать WAV".
Держите 1-2 заранее протестированных промта как запасные — на сцене LLM может закапризничать.

## Риски и запасные варианты
- **LLM отдаёт битый JSON.** → Валидатор + 1 авто-ретрай + запасной захардкоженный трек по кнопке "Пример".
- **Семплы не грузятся / лицензия.** → Дефолт на чистые синты Tone.js (работают без ассетов). Семпл-киты — бонус.
- **Латентность LLM убивает демо.** → Стриминг не нужен; покажите спиннер, держите короткие промты, кешируйте
  последний удачный ответ.
- **Не хватает времени на правки.** → Режим правок = тот же `/generate` с текущим JSON в контексте, отдельного
  кода почти нет; режем UI-фичи, а не цикл prompt→JSON→звук.
- **Web MIDI / браузеры.** → Не нужен: Tone.js играет через WebAudio, работает в Chrome/Firefox/Safari.

## Растяжка (если останется время)
- Экспорт WAV (Tone.Offline).
- MCP-обёртка: тот же `/generate` как MCP-tool, чтобы "LLM передаёт JSON через MCP" звучало на демо буквально.
- Больше звуков, простые эффекты (reverb/delay), шэринг трека по ссылке (Song JSON в URL).

## Что НЕ делаем на хакатоне
- openDAW / WASM / OPFS / собственный DSP.
- Реальный многопользовательский режим, аккаунты, БД (JSON в памяти/URL достаточно).
- Продакшн-каталог лицензированных семплов (только пара чистых наборов для демо).
