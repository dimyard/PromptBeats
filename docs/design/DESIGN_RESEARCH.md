# UX/UI-ресерч PromptBeats

## Рамка продукта

PromptBeats на хакатоне не должен быть полноценной DAW. Это быстрый и понятный творческий цикл:

1. Пользователь пишет музыкальный запрос обычным языком.
2. Приложение превращает запрос в `Song JSON`.
3. Tone.js проигрывает результат в браузере.
4. Пользователь правит трек через чат.
5. Интерфейс явно показывает, что музыка изменилась.

Главная задача UX: за первые 10 секунд объяснить цикл "написал → услышал → поправил". Экран должен ощущаться как компактная музыкальная рабочая поверхность, а не лендинг и не клон профессиональной DAW.

## Пользователь

Основной пользователь: судья хакатона или начинающий создатель, у которого есть идея, но он не хочет открывать piano roll.

Второй пользователь: продюсерски мыслящий человек, которому важно увидеть, что под капотом есть структура, а не просто магическое поле ввода.

Вывод для дизайна: промпт должен быть главным входом, но рядом всегда нужно показывать музыкальную структуру: BPM, тональность, количество тактов, дорожки, паттерны.

## Референсы и выводы

### Suno

Паттерн: простой старт через текстовый запрос, затем более глубокие правки. Suno продаёт идею AI-музыки как "напиши запрос, получи песню, редактируй дальше".

Что забрать:

- Первый сценарий должен быть очевидным: одно поле ввода и сильное действие "создать/отправить".
- Продвинутые настройки не должны мешать MVP.
- История сообщений важна: она показывает творческие итерации.

Источник: https://suno.com/

### Soundtrap

Паттерн: браузерная студия для начинающих с битмейкером, сэмплером, синтами, MIDI-редактором, эффектами и звуковыми паками.

Что забрать:

- Называть музыкальные объекты простыми словами: барабаны, бас, пэд, лид.
- Использовать знакомые DAW-элементы: Play, Stop, BPM, список дорожек, сетка паттерна.
- Не делать сырой JSON главным интерфейсом.

Источник: https://www.soundtrap.com/musicmakers

### Audiotool

Паттерн: браузерная студия с дорожками, модульными устройствами, микшером, MIDI и совместной работой. Ценность в том, что структура видна на одном экране.

Что забрать:

- One-screen studio layout подходит для демо.
- Каждая дорожка должна иметь компактную полосу: инструмент, звук, плотность паттерна, mute/solo.
- Простая grid-визуализация доказывает, что приложение генерирует настоящие музыкальные данные.

Источник: https://www.audiotool.com/

### Ableton Live

Паттерн: Session View для быстрых идей и Arrangement View для детальной раскладки. Для нашего MVP важнее ощущение лупа и клипов, а не длинный таймлайн.

Что забрать:

- Интерфейс ближе к Session View: loop-based, быстрый, паттерн-ориентированный.
- Нужны клипы/сетки, а не тяжёлая горизонтальная аранжировка.
- Transport должен быть всегда доступен и предсказуем.

Источник: https://www.ableton.com/en/live/what-is-live/

### Tone.js

Паттерн/ограничение: Tone.js хорошо подходит для интерактивной музыки в браузере, но запуск аудио требует пользовательского жеста.

Что забрать:

- Play должен выглядеть как главная кнопка, которая "включает звук".
- Нужны понятные состояния: готово, играет, генерируется, ошибка.
- Для MVP достаточно 8 тактов, 16 шагов на такт для ударных и компактных note blocks для мелодии.

Источник: https://tonejs.github.io/

## UX-принципы

1. Показывать цикл, а не механику.
   Пользователь должен понять "я прошу, оно меняется, я слышу" без инструкции.

2. Делать музыку инспектируемой.
   Судьям нужно видимое доказательство: дорожки, шаги, ноты, BPM, тональность, изменения после правки.

3. Чат = творческий лог команд.
   История сообщений показывает, как трек эволюционировал.

4. Брать DAW-метафоры, но упрощать.
   Нужны transport, track lanes, step grid, meters, sound labels. Не нужны роутинг, плагины, automation editor.

5. Надёжность демо важнее плотности фич.
   Empty/loading/error states важнее дополнительных настроек.

## Рекомендуемый экран

### Desktop

Один экран из трёх зон:

- Левая панель, 32-36% ширины: история чата и composer.
- Центральная панель, 44-50% ширины: плеер, визуализация, список дорожек, step-grid.
- Правая панель, 18-22% ширины: инспектор `Song JSON`, статус валидации, демо-промпты или payload preview.

Если времени мало, правую панель можно сделать сворачиваемой.

### Верхняя зона

Нужно показать:

- Название `PromptBeats`.
- Статус: Mock / Connected / Generating / Ready / Error.
- Transport: Play, Stop, Loop.
- BPM, key, bars.
- Export WAV как disabled/stretch.

### Чат

Структура:

- Сообщения пользователя.
- Короткие ответы системы: "Собран трек: 75 BPM, A minor, 4 дорожки".
- Composer снизу.
- Быстрые демо-промпты:
  - "Спокойный lo-fi бит, 75 BPM, минор"
  - "Добавь мягкий pad на фоне"
  - "Ускорь до 90 и сделай бас громче"

### Студия

Структура:

- Название текущего трека и метаданные.
- Большая аудио-визуализация: waveform, equalizer или beat pulse.
- Список дорожек:
  - Drums: 16-step circles.
  - Bass/Pad/Lead: note blocks по сетке.
  - Sound label из каталога: `lofi_kit`, `sine_bass`, `soft_pad`.
- Track controls: mute, solo, level meter.

### Инспектор

Для судей и команды:

- Summary-чипы: version, bpm, key, bars, tracks.
- Collapsible JSON preview.
- Validation state: schema valid, sounds from catalog.
- Payload preview: `{ prompt, currentSong }`.

## Визуальное направление

Стиль: "компактная AI beat lab".

Интерфейс должен быть музыкальным, тактильным и современным, но читаться быстро под демо-стрессом.

Палитра:

- База: near-black `#101113`, graphite `#1A1D21`, panel `#23272D`.
- Текст: soft white `#F5F2EA`, muted gray `#AEB4BD`.
- Акценты:
  - Lime `#B7FF5A` для play/active steps.
  - Cyan `#35D7FF` для AI/generation.
  - Coral `#FF6B5F` для edits/errors.
  - Amber `#FFC857` для selected/current beat.

Важно: избегать типичного фиолетово-синего AI-dashboard вида. Здесь музыка, поэтому должны работать ритм, meters и цветовые идентичности дорожек.

Типографика:

- Чистый sans-serif для интерфейса.
- Mono только для JSON/payload.
- Заголовки компактные: это инструмент, не лендинг.

Motion:

- Beat pulse при Play.
- Движущийся step cursor.
- Generating state через equalizer/wave shimmer.
- Без декоративных blob/orb-фонов.

## MVP-компоненты для Claude Design

Обязательно:

- App shell без лендинга.
- Chat history + composer.
- Transport: Play, Stop, BPM, key, bars.
- Track grid из sample `Song JSON`.
- States: ready, playing, generating, error.
- Demo prompt chips.
- Mobile responsive: чат над студией, inspector collapsed.

Можно добавить:

- Animated playhead.
- Мини-waveform/spectrum.
- JSON inspector.
- Track mute/solo.
- Diff summary после правки.

Не делать:

- Полный piano roll.
- Аккаунты/profile/feed.
- Полный микшер и plugin racks.
- Сложный sample browser.
- Marketing hero.

## Нарратив демо

Экран по умолчанию уже должен выглядеть живым: sample song loaded.

Судья видит:

- Слева историю запросов.
- В центре готовый beat sketch.
- Несколько дорожек с видимыми паттернами.
- Справа контракт `Song JSON`.

Затем демо-промпт меняет данные, а сетка дорожек заметно обновляется. Это главный UX-выигрыш.
