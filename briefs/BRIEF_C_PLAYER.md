`github.com/dimyard/PromptBeats`. Твоя роль — **Human C: браузерный аудио-движок на Tone.js**.

## Что за проект
Веб-морда с чатом → пожелание → LLM превращает его в **Song JSON** → **твой движок** проигрывает этот JSON в
браузере семплами/синтами. Правки трека приходят как новый Song JSON — движок пересобирается.

## Sampler engine — твоя зона ответственности
В этом проекте C отвечает не только за общий transport/player, но и за sampler/kit engine для дорожек
`instrument: "sampler"`.

Обязано работать:
- `sound: "lofi_kit" | "house_kit" | "trap_kit"` выбирает характер кита;
- `DRUM_NOTE_MAP` из `CONTRACTS.md` соблюдён: `C2=kick`, `D2=snare`, `F#2=closed hat`, `A#2=open hat` и т.д.;
- `gain` и `muted` применяются на уровне дорожки при каждом `load(song)`;
- ручные правки A в `track.events` сразу слышны после повторного `player.load(updatedSong)`;
- неизвестная drum note не роняет плеер: используй fallback и/или событие `error`.

Для MVP можно оставаться на синтезированных китах Tone.js, если реальные сэмплы не успеваем или нет чистой лицензии.
Но киты должны отличаться на слух: например, `lofi_kit` мягче/грязнее, `house_kit` плотнее и ярче, `trap_kit` суше
с коротким snare/clap и заметными hats.

## Твоя задача (по приоритету)
1. **Качество звука.** Улучшить синты и особенно ударные. Сейчас киты — синтезированный шум/мембрана; по возможности
   перевести `lofi_kit`/`house_kit`/`trap_kit` на реальные сэмплы через `Tone.Sampler` (см. «Лицензии сэмплов» ниже),
   различить киты между собой. Соблюдай `DRUM_NOTE_MAP` из контракта (C2=kick, D2=snare, F#2=closed hat, ...).
2. **Микс.** Аккуратные уровни по дорожкам (`gain`), лёгкая мастер-цепочка (например, reverb/limiter), чтобы не
   клиппило и звучало «слитно».
3. **Поддержка ручных контролов A.** Фронт будет менять `track.muted`, `track.gain`, `track.sound` и `track.events`,
   затем вызывать `player.load(updatedSong)`. Проверь, что эти изменения применяются предсказуемо и быстро.
4. **Надёжность пересборки.** Убедись, что многократный `load()` при правках не даёт наложений, утечек нот и щелчков.
   Неизвестный `sound` → fallback + событие `error` (уже есть, проверь).
5. **Растяжка — `exportWav(song)`.** Офлайн-рендер через `Tone.Offline`, вернуть `Blob` (WAV). Это фича «Скачать» на демо.
6. (Опц.) Простые эффекты по `role`/дорожке: delay на lead, reverb на pad.

## Definition of Done
- `createPlayer().load(sample-song.json)` играет приятный звук; `play/stop` работают; плейхед (`step`) идёт по сетке.
- Повторные `load()` с разными песнями переключают трек чисто, без артефактов.
- Sampler/kit дорожки поддерживают ручные изменения `gain`, `muted`, `sound`, `events` через повторный `load(song)`.
- Реализован `exportWav()` (или явно отмечено в журнале, если не успел).
- Контракт не нарушен. Запись в `IMPLEMENTATION_LOG.md` добавлена.

## Лицензии сэмплов (не игнорировать)
Если добавляешь сэмплы — только лицензионно чистые (CC0 / public domain, напр. с freesound под CC0). Клади в
`frontend/public/samples/…`, источник и лицензию зафиксируй в журнале. Сомневаешься — оставайся на синтезированных китах.

## Как проверить (без бэка)
Временно скорми плееру `sample-song.json` (импортом или fetch из `public/`). Проверь в Chrome, Firefox, Safari — звук
идёт через WebAudio, Web MIDI не нужен. Первый `play()` должен быть по клику (требование `Tone.start()`).
Прогони правку вручную: вызови `load()` со вторым, отличающимся Song (другой bpm/дорожки) — переключение должно быть чистым.
